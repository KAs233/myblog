# 登录及接口鉴权逻辑

首先用户分为基本用户、运维人员两种，均存在Person表，基本用户会多存一张user表。登录时，对两种用户会有差异。

登录时，先调用login/door、login/room接口，对应Login、Login2方法，分别用于校验账号密码（若基本用户则返回其租户身份信息；密码存储采用基础部分+固定字符+guidId编码的MD5加密）、获取token凭证（采用时间+用户ID的SHA加密），并存于redis缓存中。

后端位置：src\domain\permission\Application\...\Impl\LoginService.cs

前端会将token存放于请求的header中一同进行请求。后续接口通过AuthMiddleware中间件来进行接口鉴权，除了白名单之外的接口均需要进行token有无的验证，再讲传入的token放置于静态字典中，在各接口的使用中通过AuthHelper获取用户信息时会验证静态字典与redis中token是否一致。

# 前端NestedTable组件常用说明

NestedTable为前端已封装好的列表组件，方便快速搭建页面。其包含了很多属性，在实际使用时，进行调用传参，可实现不同页面的不要需要。默认加载新增、删除两个按钮，以及双击行等调用上级事件。以下会按照功能来列举一些常用的属性。

更多属性及配置方法，在需要的时候阅读相关组件页面即可。在VSCode中可直接Ctrl + P快速定位。

位置：src\components\Table\NestedTable.vue

props:{
    //基础列表

    //左边功能列显示

    leftColumns: {
      type: Object,

        return { expand: false, select: false, order: false }

序列化属性在后端哪里处理的（如何在返回时将字段名首字母序列化为小写）？

ABP框架在做ToJson处理时，默认将属性按照驼峰命名法进行序列化。当然这一过程也可以进行自定义配置。

前端api字段替换，及端口替换过程是如何执行的？

在vue.config.js中进行相关配置。通过vue-cli3设置代理解决webapi中的跨域。所以可通过前端端口访问到后端端口api。
