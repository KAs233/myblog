# 概述

很多时候我们只是在用npm，去不知道npm到底是什么。我总觉得看待问题应该通过WWH（WHAT-WHY-HOW）的方式，所以我们先来看一看npm到底是什么吧。

# npm是什么

npm的全称是Node Package Manager，其实知道了全称就不难猜出它大概是个什么东西—运行于Node.js的包管理器。

在官网的描述[^1]中，npm是世界上最大的软件代码仓库。开源开发者可以从任何位置使用npm去分享借鉴包。

它主要由三部分功能组成：

  • 网站

    可以使用其官方网站 (opens new window)去发现包、设置配置文件和管理npm体验的其他方面。

  • 命令行接口(CLI)

    CLI从终端运行，这是绝大多数开发者与npm进行交互的方式。

  • 代码仓库

    代码仓库是指其本身是一个大型的公共数据库，包含了JavaScript软件及相关的应用。

[^1]: 官网的描述：npm is the world's largest software registry. Open source developers from every continent use npm to share and borrow packages, and many organizations use npm to manage private development as well.
